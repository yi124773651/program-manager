# 版本与迁移规则

## 应用版本

应用发布版本必须保持以下文件一致：

- `package.json`
- `src-tauri/Cargo.toml`
- `src-tauri/tauri.conf.json`

发布前执行：

```bash
npm run check:versions
```

## 数据结构版本

数据结构版本使用各数据文件中的 `schemaVersion` 表示，不直接等同应用发布版本。

当前约定：

- `config.json` 继续保留业务配置中的 `version` 字段，用于兼容旧配置。
- 新增独立数据文件使用统一外壳：`schemaVersion`、`updatedAt`、`data`。
- 迁移执行记录保存在 `%APPDATA%/program-manager/migrations.json`。

## 迁移原则

1. 迁移前必须备份。
2. 迁移失败不得破坏旧数据。
3. 已完成的迁移必须写入迁移记录，避免重复执行。
4. `localStorage` 旧数据由前端读取后提交给后端落盘，后端不直接读取 WebView 存储。
